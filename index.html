1. Difference between var and let:

    var is function-scoped, meaning it is accessible within the function it is declared in, regardless of block boundaries (such as loops or conditionals).
    let is block-scoped, meaning it is only accessible within the block it is declared in (like loops or conditionals). It has a more limited scope than var and helps avoid certain types of bugs caused by variable hoisting and unintended reassignment.

2. Difference between var and const:

    var allows reassignment and its value can be changed.
    also has block scope like let, but it is a constant and cannot be reassigned once a value is assigned to it.

3. Difference between let and const:

    let allows reassignment of its value within its scope.
    const is a constant and its value cannot be changed once assigned.

4. Hoisting:

Hoisting is a JavaScript mechanism where variable and function declarations are moved to the top of their containing scope during the compilation phase. However, only the declarations are hoisted, not the initializations or assignments.
